Methodology

This section outlines the complete workflow followed in the GEONEX project, from data collection to AI-based classification and report generation.

Project Overview

The GEONEX system is designed to classify satellite images into land cover categories using deep learning. The workflow includes data collection, preprocessing, model training, cloud storage integration, and automated report generation.

Data Collection

Satellite images were collected and organized based on location and date. The dataset was structured into predefined land cover classes for supervised learning.

Data Preprocessing

Images were resized, normalized, and labeled properly. The dataset was divided into training and validation sets to ensure proper model evaluation.

Model development

A Convolutional Neural Network (CNN) model was developed and trained to classify images into different land cover categories. The model performance was evaluated using accuracy metrics.

Model Training and Evaluation

The model was trained on labeled satellite images. Validation techniques were used to monitor performance and reduce overfitting.

Cloud Integration

Classified images were automatically uploaded to cloud storage and organized into folders based on predicted classes and location.

Web Application Development

A Flask-based web application was developed to allow users to select location and date range, generate reports, and view classified image folders.

Report Generation

Based on user input, the system filters images by location and date, groups them into classified folders, and displays structured results for analysis.

Testing and Validation

The complete system was tested to ensure accurate classification, proper cloud storage handling, and smooth report generation workflow.

Final Deployment

The GEONEX system was deployed with integrated AI model, cloud storage, and web interface to provide automated geospatial land cover analysis.
